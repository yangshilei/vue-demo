答案为纯命令敲击答案，使用TortoiseGit可以忽略
一.拉取项目代码：
	答案：git pull 项目地址
	
二.拉取完项目默认是master分支：切换到开发人员的devloper分支；
	答案：1.先查看项目所有分支列表：git branch -a 
		  2.git checkout devloper(分支名输入开头，按tab键可以自动补全)
		  3.检查是否成功切换到devloper分支，也就是检查当前分支：git branch
		  
三.更新devloper分支最新代码
	答案：git pull
	
四：修改或删除一个文件（模拟在原来写好的代码文件中写新的代码或删除无用文件），并提交代码
	答案：1.先提交代码到本地：git commit "提交说明"
		  2.再推送到远程仓库：git push 
		  
五: 新增一个代码文件，并提交代码;步骤只比直接提交多一步添加文件操作；
	答案： 1.git add "文件名"
		   2.git commit "提交说明"
		   3.git push 
		   
六：代码和别人冲突：
	答案1：用webstorm，内嵌git工具，可以直接在当前分支和其它分支比较修改自己需要的代码；
	答案：1.比价冲突的代码，避免覆盖别人写好的代码
		  2.将自己的代码备份
		  3.手动处理冲突部分代码
		  4.处理完提交代码即可

七：将dev开发分支代码提交到test测试分支；
	答案1：用webstorm，内嵌git工具，可以直接在当前分支和其它分支比较，并合并，省事很多；
	答案2：1.建两个文件夹：一个放dev分支代码，一个放test分支代码
		  2.使用比较工具（Compare）对两个分支代码进行比较
		  3.将dev模块自己写的代码合并到test分支
		  4.合并完提交到test分支
